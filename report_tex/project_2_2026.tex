\begin{abstract}
本项目围绕 CIFAR-10 图像分类任务以及 Batch Normalization（BN）对优化过程的影响展开研究。我们使用 PyTorch 实现了三种 VGG 风格模型：基础版 VGG-A、加入 BN 的 VGG-A，以及加入 Dropout 的 VGG-A。实验中比较了不同优化器和学习率，并分别进行了小规模探索实验与全量数据训练，同时结合 loss landscape 与梯度诊断来分析 BN 对训练行为的影响。在此基础上，我们进一步研究了模型宽度、激活函数以及 loss/regularization 对性能的影响。最终全量训练中，使用 Adam 且学习率为 $10^{-4}$ 的普通 VGG-A 取得了最高 $76.83\%$ 的测试精度，对应 test error 为 $23.17\%$；相同设置下，VGG-A + BN 的最高测试精度为 $74.60\%$，对应 test error 为 $25.40\%$。因此，就本次作业的最终结果而言，加入 BN 并没有优于普通 VGG-A。
\end{abstract}

\section{引言}
CIFAR-10 是图像分类中非常经典的基准数据集，包含 10 个类别共 60,000 张 $32 \times 32$ 的彩色图像。由于图像分辨率较低，它非常适合作为研究网络结构设计与优化行为的实验平台。本次项目的第一个目标是在 CIFAR-10 上训练一个包含卷积层、池化层、激活函数和全连接层的卷积神经网络；第二个目标是研究 Batch Normalization 对训练动态与优化过程的影响。

围绕这一目标，我们实现了 VGG 风格网络，并将 Batch Normalization 作为主要扩展组件，同时保留 Dropout 作为补充对比；此外还比较了 \texttt{torch.optim} 中的多种优化器，并对 loss landscape 与梯度变化进行可视化分析。随后我们进一步加入了不同宽度配置、不同激活函数以及不同 loss/regularization 的对比实验，使整个研究同时覆盖结构设计、优化策略和训练解释三个层面。

\section{数据集与实验设置}
\subsection{数据集}
本项目使用 CIFAR-10 数据集。完整数据集包含 50,000 张训练图像和 10,000 张测试图像。在正式全量训练之前，我们先用两级小规模实验完成方案比较，以提高迭代效率：
\begin{itemize}
    \item 5,000/1,000 的训练/测试子集用于主线方案比较，例如 BN 行为和优化器选择；
    \item 3,000/1,000 的更轻量训练/测试子集用于结构、激活函数与 loss/regularization 的快速对比。
\end{itemize}
最终结论统一以全量 CIFAR-10 训练结果为准。

\subsection{模型结构}
我们在 \texttt{codes/VGG\_BatchNorm/models/vgg.py} 中实现了三种 VGG 风格模型：
\begin{itemize}
    \item \textbf{VGG-A}：基础模型；
    \item \textbf{VGG-A + BN}：在每个卷积层后加入 BN，并在分类器隐藏层激活前加入 BN；
    \item \textbf{VGG-A + Dropout}：在分类器部分加入 Dropout。
\end{itemize}

以上模型都包含卷积层、最大池化层、ReLU 类激活函数和全连接层，并构成了本文后续比较实验的统一基础。为了进一步分析模型容量与非线性设计对训练表现的影响，我们将 VGG-A 扩展成了可配置版本，支持修改：
\begin{itemize}
    \item 卷积通道宽度倍率（width multiplier）；
    \item 分类器隐藏层维度；
    \item 激活函数种类（ReLU、LeakyReLU、ELU）。
\end{itemize}

\subsection{训练流程}
所有实验均基于 PyTorch 实现，主训练与分析脚本为 \texttt{codes/VGG\_BatchNorm/VGG\_Loss\_Landscape.py}。我们将原始骨架代码扩展为一个可复用的实验运行器，支持：
\begin{itemize}
    \item 模型与优化器对比；
    \item 自动保存训练曲线与指标；
    \item loss landscape 与梯度诊断；
    \item 长时间实验的断点续跑；
    \item 可配置的激活函数、模型宽度、隐藏层维度、loss 类型与正则化强度。
\end{itemize}

在优化策略方面，我们比较了 SGD、带 momentum 的 SGD 以及 Adam。其中，最终全量训练采用 Adam、学习率 $10^{-4}$、batch size 128、40 个 epoch；而快速比较阶段的对比实验统一采用 Adam、学习率 $10^{-4}$、3,000/1,000 的子集和 6 个 epoch，以保证实验之间可公平对比。

\subsection{小规模探索实验结果}
我们首先在 5,000 张训练图像和 1,000 张测试图像组成的小规模划分上进行实验，用于验证代码正确性并快速比较不同训练配置。表 \ref{tab:subset-results} 总结了其中具有代表性的结果。

\section{VGG-A 与 VGG-A + BN 的比较}
为了更直接地分析 BN 的作用，我们在相同骨干结构下比较了 VGG-A 与 VGG-A + BN。BN 版本通过在每个卷积层后加入 \texttt{BatchNorm2d}，并在分类器隐藏层激活前加入 \texttt{BatchNorm1d} 来实现。这样可以尽量保持网络结构与基线模型接近，使对比更加直观。

这组比较得到的结论：
\begin{itemize}
    \item 在小规模探索实验中，BN 版本取得了更高的测试子集精度，并且对学习率变化更稳健；
    \item 在最终的全量 40 epoch、Adam、学习率 $10^{-4}$ 这一设置下，普通 VGG-A 的最终测试精度略高于 BN 版本。
\end{itemize}

因此，从本次实验结果来看，BN 不是更优方案。它在小规模探索阶段曾带来一定收益，但在最终全量训练中没有超过普通 VGG-A。

\section{结构、激活与 Loss/Regularization 分析}

\section{讨论与结论}
本项目可以总结出以下几点。

\paragraph{1. 优化器选择非常重要。}
在主线实验中，Adam 和带 momentum 的 SGD 都显著强于普通 SGD，这一点在 BN 模型上尤其明显。

\paragraph{2. BN 在本次实验中没有带来更好的最终结果。}
在小规模探索阶段，BN 一度表现出更高的测试子集精度；但在最终全量训练中，VGG-A + BN 的最好测试精度只有 74.60\%，低于普通 VGG-A 的 76.83\%。因此，就这组作业实验而言，加入 BN 没有带来更好的最终性能。

\paragraph{3. 结构、激活与 loss 选择都会显著影响结果。}
实验结果表明，模型宽度并不是越大越好；ELU 在短程子集训练中明显优于 ReLU；Focal Loss 和 label smoothing 也能带来一定提升。也就是说，这些设计项都会真实影响训练表现，而不是仅仅停留在结构层面的变化。

\paragraph{4. 更好的优化行为不一定对应最高最终测试精度。}
在当前最佳全量配置下，普通 VGG-A 的最终测试精度高于 BN 版本。与此同时，从小规模比较中挑出的 ELU 和 Focal Loss 也没有在全量上超越原始基线。这说明无论是 BN 还是其它设计项，小规模结果都不能直接替代全量验证；最终配置必须结合完整训练结果来判断。

未来最值得继续尝试的方向包括：更强的数据增强、更合理的学习率调度，以及针对 BN 版本进行更长时间的专项调参。这些改进有望同时提升两类模型的效果，也可能改变普通 VGG-A 与 VGG-A + BN 之间的最终排名。
